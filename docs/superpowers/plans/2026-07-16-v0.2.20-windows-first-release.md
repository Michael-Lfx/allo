# v0.2.20 Windows-First GitHub Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to execute this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish `v0.2.20` as the first GitHub Release for this version with a signed Windows x64 updater package and a verified updater manifest.

**Architecture:** Use the repository's `release:win` CREATE workflow as the release orchestrator. It owns the version bump, native Windows release build, Tauri updater signing, `latest.json` generation, release commit and tag, pushes, GitHub Release creation, asset upload, and endpoint verification.

**Tech Stack:** Bun, PowerShell, Cargo/Tauri, Git, GitHub CLI, Tauri updater/minisign

## Global Constraints

- Release version is exactly `0.2.20`; the Git tag and GitHub Release name are exactly `v0.2.20`.
- Publish a normal public Release: not a draft and not a prerelease.
- This is a Windows-first CREATE release; macOS and Linux assets are appended later from native build machines.
- The Windows updater package must have a Tauri updater `.sig` generated with the configured private key.
- Authenticode is not enabled in the current Windows release script, so the public notes must disclose the possible SmartScreen/unknown-publisher warning.
- Do not print or commit `GH_TOKEN` or the updater private key.
- Set `core.longpaths=true` through `GIT_CONFIG_COUNT` for all release commands, because the current Codex checkpoint refs exceed the default Windows path limit.


Expected: clean synchronized `main`; `HEAD`, `origin/main`, and `v0.2.20` resolve to the release commit.

- [ ] **Step 2: Verify GitHub Release metadata and assets**

Run:

```powershell
$envFile = 'apps/desktop/signing/.env.release'
foreach ($line in Get-Content -LiteralPath $envFile) {
  $trimmed = $line.Trim()
  if ($trimmed -match '^\s*GH_TOKEN\s*=\s*(.+)$') { $env:GH_TOKEN = $Matches[1].Trim() }
  elseif ($trimmed -match '^(ghp_|github_pat_)') { $env:GH_TOKEN = $trimmed }
}
& 'C:\Program Files\GitHub CLI\gh.exe' release view v0.2.20 --repo nomifun/nomifun-tauri --json tagName,name,isDraft,isPrerelease,publishedAt,url,assets
```

Expected: tag/name `v0.2.20`, `isDraft=false`, `isPrerelease=false`, and uploaded assets named `NomiFun_0.2.20_x64-setup.exe`, `NomiFun_0.2.20_x64-setup.exe.sig`, and `latest.json`.
