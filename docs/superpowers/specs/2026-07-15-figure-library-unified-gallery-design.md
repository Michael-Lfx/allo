# 形象库统一画廊卡片设计

## 目标

将形象库中的已保存形象与“新建 / 导入形象”入口改为同一套竖版卡片规格，避免不同图片比例和名称长度造成列表视觉跳动。设计应让用户快速扫视内容、清楚辨认名称，并保持上传入口与现有资产同级。

## 布局与视觉

- 网格中的每张卡片固定为 `184px × 234px`，固定列宽而不由可用空间拉伸。
- 卡片顶部的预览区固定为 `184px × 190px`；继续使用透明图棋盘底纹，图片以 `object-contain` 居中显示，最大显示范围为预览区的 88%。
- 名称区固定为 `44px` 高，使用左右 `12px` 内边距和单行截断；不使用上分隔线，长名称不会改变卡片高度。
- “新建 / 导入形象”保留与其他卡片相同的外层尺寸，但采用截图指定的极简入口：浅色实底、无虚线边框、居中的浅灰圆形加号，以及图标正下方的单行低对比标签。它不使用普通卡片的预览区与名称栏分隔。
- 卡片使用细边框、16px 圆角和低强度悬浮阴影。编辑和删除操作仅在悬浮或键盘焦点时显现；在用标记仍保持可见。

## 实现边界

- 仅调整 `FigureLibraryPage` 中的列表布局和卡片样式；不改变形象数据、创建流程、编辑流程或删除保护。
- 为卡片规格增加源码级回归断言，确保已保存卡片和创建入口共享外层、预览区、名称区高度约束。

## 验证

- 运行形象库相关 Bun 测试。
- 运行 TypeScript 类型检查。
- 在现有桌面开发环境中检查至少一张竖向人物图、一张横向头像图和创建入口，确认三张卡片的外框、图片区与底部文字区对齐。
