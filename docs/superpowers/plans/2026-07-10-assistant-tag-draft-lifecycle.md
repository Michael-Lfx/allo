# Assistant Tag Draft Lifecycle Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Discard unfinished inline assistant-tag drafts when the assistant drawer is saved or closed, while retaining the draft on ordinary blur and showing an Enter-key hint.

**Architecture:** `AssistantEditDrawer` owns the lifecycle boundary for its two shared `AssistantTagPicker` children. It resets both picker refs before saves and local closes, then observes `editVisible` as a parent-controlled-close safety net. `AssistantTagPicker` continues to own draft state and adds a localized helper text only while its input is active.

**Tech Stack:** React 19, TypeScript, Arco Design, react-i18next, bun:test.

## Global Constraints

- Scope is limited to the assistant create/edit drawer; `SkillTagModal` retains its existing `commitOnBlur` behavior.
- Pressing Enter remains the only way an assistant-drawer tag draft creates a tag.
- A blur in the open assistant drawer keeps its text and input state unchanged.
- `settings.assistantTagAddHint` must exist in English and Simplified Chinese; refresh the generated `I18nKey` union with `bun run gen:i18n`.
- Frontend regression tests use `bun:test`: `cd ui && bun test <path>`.

import { describe, expect, test } from 'bun:test';
import { createAssistantTagDraftLifecycle } from './assistantTagDraftLifecycle';

describe('assistant tag draft lifecycle', () => {
  test('resets both drafts before saving', () => {
    const calls: string[] = [];
    const lifecycle = createAssistantTagDraftLifecycle(
      { current: { resetPendingTag: () => calls.push('audience') } },
      { current: { resetPendingTag: () => calls.push('scenario') } },
      (visible) => calls.push(`visible:${visible}`),
      () => calls.push('save')
    );

    lifecycle.handleDrawerSave();

    expect(calls).toEqual(['audience', 'scenario', 'save']);
  });
});

- [ ] **Step 3: Centralize draft cleanup in a testable lifecycle helper**

Replace the React import with one that includes `useCallback`, and import the picker handle type:

```ts
import AssistantTagPicker, { type AssistantTagPickerHandle } from './AssistantTagPicker';
import { createAssistantTagDraftLifecycle } from './assistantTagDraftLifecycle';
import React, { useEffect, useMemo, useRef, useState } from 'react';
```

Place these declarations immediately after the existing refs and drawer local state:

```ts
  const audiencePickerRef = useRef<AssistantTagPickerHandle>(null);
  const scenarioPickerRef = useRef<AssistantTagPickerHandle>(null);

  const { resetPendingTagDrafts, closeDrawer, handleDrawerSave } = useMemo(
    () => createAssistantTagDraftLifecycle(audiencePickerRef, scenarioPickerRef, setEditVisible, handleSave),
    [handleSave, setEditVisible]
  );

  useEffect(() => {
    if (!editVisible) {
      resetPendingTagDrafts();
    }
  }, [editVisible, resetPendingTagDrafts]);
```

Pass `ref={audiencePickerRef}` and `ref={scenarioPickerRef}` to the audience and scenario picker instances. Use `closeDrawer` for the header close control, `Drawer.onCancel`, and the Cancel button. Use `handleDrawerSave` for the Create/Save button.
